Git — распределённая система контроля версий для отслеживания изменений в файлах проекта.

Основные понятия

Репозиторий (repository)** — хранилище проекта со всей историей изменений. Бывает:
локальный** (на компьютере разработчика);
удалённый** (на сервере: GitHub, GitLab и т. д.).

Ветка (branch)** — независимая линия разработки: последовательность коммитов. Позволяет изолировать изменения (новые функции, исправления) от основной ветки (`main`/`master`).

Коммит (commit)** — снимок состояния проекта в определённый момент. Содержит:
* ссылки на файлы;
* метаданные (автор, дата);
* сообщение о сути изменений.

Тег (tag)** — метка для важного коммита (например, релиз версии). Типы:
легковесные** — простая ссылка на коммит;
аннотированные** — объект с автором, датой, сообщением, может быть подписан.

Основные команды Git (кратко)

 `git init` — создать локальный репозиторий.
 `git clone <URL>` — клонировать удалённый репозиторий.
 `git status` — показать состояние файлов (изменённые, новые и т. д.).
`git add <файл>` или `git add .` — добавить файлы в индекс (к коммиту).
`git commit -m "Сообщение"` — зафиксировать изменения.
git log` (с `--oneline` — компактно) — посмотреть историю коммитов.
 `git diff` — увидеть различия между версиями.
`git branch` — показать ветки; `git branch <имя>` — создать ветку.
`git checkout <ветка>` — переключиться на ветку; `git checkout -b <ветка>` — создать и переключиться.
`git merge <ветка>` — слить изменения из указанной ветки.
git pull` — получить и слить изменения из удалённого репозитория.
git push` — отправить локальные коммиты на сервер.
git remote add origin <URL>` — связать локальный и удалённый репозитории.
 `git reset --hard` — откатить изменения (осторожно!).
`git revert <коммит>` — отменить изменения коммита без удаления истории.
 `git stash` — временно сохранить незакоммиченные изменения.
`git tag <имя>` — создать тег на текущем коммите.
